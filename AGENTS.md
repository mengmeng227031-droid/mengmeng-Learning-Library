# AGENTS.md

## 项目定位

`mengmengteach` 是「蒙蒙学习库」Windows 客户端 / Web MVP 原型项目。生成任何页面、组件、海报或客户端界面时，必须优先参考本项目的品牌视觉手册。

## 品牌资产

- 完整视觉手册：`brand/mengmeng-visual-manual.png`
- Logo 裁切图：`assets/brand/brand-logo.png`
- 吉祥物横幅：`assets/brand/brand-mascots.png`
- 品牌封面参考：`assets/brand/brand-cover.png`
- 品牌理念图标参考：`assets/brand/brand-concept-icons.png`

## 视觉规则

- 主色必须来自视觉手册：
  - 蓝色 `#4D8BFF`
  - 粉色 `#FF6EC7`
  - 黄色 `#FFC83D`
  - 青色 `#37D2C6`
  - 紫色 `#7B61FF`
- 辅助色优先使用：
  - 橙色 `#FF8A42`
  - 草绿 `#A0E75A`
  - 珊瑚粉 `#FF6F91`
  - 天蓝 `#6AD4FF`
  - 浅紫 `#B8A0FF`
- 中性色：
  - 标题 `#333333`
  - 正文 `#666666`
  - 弱文字 `#999999`
  - 边框 `#CCCCCC`
  - 背景 `#F5F7FA`
- 页面气质：儿童学习、快乐探索、智慧成长，整体要轻盈、明亮、圆润、干净。
- 允许使用星星、爱心、音符、星球、弧线、圆形、圆角方形、三角形、叶片等装饰元素。
- 吉祥物是核心视觉信号，首页和重要模块应出现猴子、猫、狗、河马、大象、老鼠这一组角色中的至少一种形象或其裁切资产。

## 字体规则

- 中文优先：`Microsoft YaHei`, `PingFang SC`, `思源黑体`。
- 英文优先：`Nunito`, `Poppins`, `Segoe UI`。
- 标题要圆润有重量，正文保持清晰易读。

## 编码规则

- 所有 HTML、CSS、JS、JSON、Markdown 等文本文件统一使用 UTF-8 编码保存。
- 页面可见中文、数据文件中的中文、脚本生成的中文必须在浏览器和 Node 读取时保持正常中文，不允许出现乱码或 `???`。
- 在 Windows PowerShell 中生成含中文内容的文件时，必须额外验证 UTF-8 读取结果；不要直接相信终端显示，因为终端代码页可能把正常 UTF-8 显示成乱码。
- 如需批量生成中文数据，优先使用已有 UTF-8 文件、`apply_patch` 或明确指定 UTF-8 的脚本写入，并用 `node`/浏览器读取验证中文。

## 交互与页面规范

- 首页优先做真实可用界面，不做营销落地页。
- 页面模块应数据驱动，课程卡片、导航、快捷入口、最近学习记录不要直接写死在 HTML 里。
- Windows 客户端 MVP 阶段保持静态 HTML/CSS/JS 可直接打开。
- 后续动态化时，将 `script.js` 中的数据数组替换为 API 数据，不要改变视觉规则。
- 项目最终需要支持学生账号登录、按账号动态填充今日推荐/学习内容，并接入数据库；后续每个细小功能都要为用户身份、API 数据源、数据库字段和权限边界预留清晰结构，不写死只适用于单用户的实现。

## 禁止项

- 不使用深色、沉重、商务感强的视觉。
- 不使用一整页单色渐变或廉价 AI 感强的占位图。
- 不新增 `万事通`、`一对一学习` 模块，除非用户明确要求恢复。
- 不把品牌手册图片从项目中删除或改名。
